# 客户应用官网

正式域名：`https://www.ruscny.net`

## 公开页面

| 路径 | 内容 |
| --- | --- |
| `/` | 中俄双语产品官网：首页、核心功能、AI 会议纪要、账号安全、多人会议、使用流程和下载状态 |
| `/account` | 中俄双语正式账号注册/登录、资料头像、个人偏好、密码与登录设备管理；`/register`、`/login` 为直接入口 |
| `/privacy` | 中俄双语隐私政策发布候选文本 |
| `/terms` | 中俄双语用户协议发布候选文本 |
| `/join` / `/join/:token` | 临时用户浏览器参会与邀请链接入口 |
| `/robots.txt` | 禁止搜索引擎抓取账号、管理后台和浏览器参会等非公开内容页 |
| `/sitemap.xml` | 官网、隐私政策和用户协议索引 |
| `/og.png` | 1200×630 官网分享卡片 |

官网和 API 由同一个 Fastify 服务交付。API 合同继续使用 `/v1/*`，Socket.IO 使用 `/socket.io/*`，管理后台使用 `/admin`，因此生产反向代理不得把根站点部署成与 API 完全分离且无法同源访问的静态项目。

## 语言与内容

- 默认根据浏览器语言选择中文或俄文；其他语言回落中文。
- 用户选择保存在浏览器本地，仅用于界面偏好。
- 首页不展示管理后台入口。
- 首页导航、首屏、下载区和页脚均提供正式账号入口；注册资料包含显示名称、可选公司、邮箱、密码和主要语言，不显示或选择账号类型。
- 注册和登录直接调用同源 `/v1/auth/register`、`/v1/auth/login`。所有正式账号能力相同；任一注册用户创建房间后，服务端以 `Conversation.ownerId` 和该会议的 `Participant.role=HOST` 确认其主持人身份。
- Access/Refresh Token 只保存在当前标签页的 `sessionStorage`，不写入 URL 或持久化存储；非秘密设备标识保存在本地以维持设备会话。页面刷新时使用旋转 Refresh Token 恢复，退出时调用服务端 logout 后清除本地会话。
- 正式账号的头像主题、主要语言、界面语言、译文自动播放和播放速度由服务端保存；手机 App 登录或恢复会话时应用账号值，手机端修改后同步回账号。网页“密码与安全”页可以查看设备状态并远程撤销其他设备。
- `/account` 使用 `no-store`、限制性 CSP 和 `noindex`。页面不提供完整 Web 工作区，注册后需在手机 App 使用同一账号完成好友、会议和历史管理。
- App Store 与 Google Play 尚未发布时，下载卡片明确显示“即将开放”，不提供虚假链接。
- 临时参会入口直接进入现有 `/join` 页面。

## 发布阻断项

正式公开前仍必须补齐：

1. 运营主体、地址、客服和隐私联系方式。
2. 数据存储国家/地域、第三方清单和具体保留期限。
3. 收费、免费额度、退款、适用法律和争议解决条款。
4. App Store、Google Play 或正式受控下载地址。
5. 法律顾问对隐私政策和用户协议的审核结果。
6. `www.ruscny.net` 的 DNS、TLS、同源反向代理和公网 smoke test。

候选法律页面会明确显示尚未补齐的信息，不能把候选文本表述为已经正式生效。

## 验证

运行 `npm run test:web` 检查中俄文案完整性、账号字段/API 接线、令牌存储边界、不安全 DOM sink、公开链接、社交卡片和浏览器参会安全回归。API 的 `app-public` 测试同时验证首页、账号页及别名、法律页面、静态资源、`no-store`、CSP、robots 和 sitemap 路由。
