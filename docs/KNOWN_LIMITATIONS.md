# 当前限制与发布阻断项

更新时间：2026-07-21

## 1. 当前交付状态

本仓库是多人会议版本的工程交付，不等同于已经上架的生产 App。2026-07-19 当前工作区通过客户官网账号、邮箱激活、邮件重置密码与 H5 Web 回归 17/17、Prisma generate/validate、后端 build/typecheck、236/236 单元测试、Flutter analyzer、59/59 Flutter 测试和生产依赖审计（0 个已知漏洞）。当前工作区共有 21 个 PostgreSQL 迁移；现有 GitHub CI 证据只覆盖当时的前 16 个迁移、13/13 API/Socket 集成测试、生产 Docker 镜像、51/51 Flutter 测试、Android debug APK 和 iOS Simulator App 构建。最新 5 个迁移、新增邮箱激活/重置的 PostgreSQL 集成场景与移动端流程仍需提交后 CI 复验。API、Socket 和 App Link 配置指向最终域名 `www.ruscny.net`，但正式商店 App 仍未验收。当前仍缺 Apple/Google 发布账号、release 签名材料、真机矩阵、Redis 两个 API 副本的跨实例故障演练、阿里云生产凭据及经过受控收件箱验证的生产邮件投递证据。

## 2. 阿里云能力变化

原始指令指定 Gummy Realtime，但截至 2026-07-18，阿里云官方模型目录已把 Gummy 列为即将下线能力，且官方页面对中俄方向的描述存在差异。新生产链路默认使用可配置的 Qwen3-ASR-Flash → Qwen-MT → Qwen3-TTS/CosyVoice。Gummy 仅在目标账号验证双方向、配额和生命周期后才能作为可选适配器。

模型可用地域、价格、限流、音色和生命周期会变化。本文档中的模型名不是永久保证；每次发布/切换必须核对官方文档、目标业务空间和真实音频。

## 3. 第一版产品限制

这些是明确范围，不属于缺陷：

- 只支持中文与俄语，不支持其他语言。
- 必须手动选择输入语言，不做自动语言识别。
- 按住说话、松开后处理，不做连续同声传译。
- 每台设备明确绑定一个 Participant，能可靠保留“谁说了什么”；当前不支持多人共用同一麦克风时的自动说话人分离/声纹识别，也不支持重叠发言的自动拆分。
- Android 已实现好友一对一语音/视频选择的本地版本；尚不支持 iOS 好友视频、多人视频、Viewer、完整 CRM 或复杂会员付费。
- 默认不长期保存完整原始录音，不提供录音回放。
- 已提供按单次 `conversationId` 隔离的结构化会议纪要保存、读取以及阿里云百炼通义千问 AI 整理；模型输出必须携带发言 sequence 依据并由服务端复验，主持人确认后才分发。当前仍需使用真实中俄商务会议完成模型质量、长会议容量、费用、地域和人工复核验收；AI 结果不能视为未经确认的事实。
- 已提供主持人逐人选择并发送会议纪要的接口与 App 操作；当前记录邮件供应商“已受理/失败”，尚未接入 delivered、bounced、complained webhook，因此不能把 `SENT` 解释为收件人最终收到。生产上线还需验证发信子域、限流、退信、垃圾邮件和中俄邮件显示。
- APNs/FCM 来电推送仍仅预留；App 前台会用 Socket.IO 与 2 秒 REST 恢复轮询收敛来电，但进程被系统冻结或杀死时不能承诺即时响铃。实时翻译本身依赖已建立的 Socket.IO/RTC 会话，不依赖推送承载音频。
- iPad 只做基础兼容，不承诺专门的大屏布局。

## 4. 技术限制

- 本地 SQLite 只是有界缓存，不能作为权限或历史的权威来源；离线状态不允许新建翻译消息长期排队。当前 SQLite 没有应用层加密，依赖 iOS/Android 应用沙箱与设备加密；退出/注销会最佳努力清空私有缓存，高风险部署应改用可验证的加密存储。
- 第一版音频通过 HTTP 整段上传，延迟会包含完整录音时长、上传、ASR、MT 和 TTS；弱网下不等同连续字幕体验。
- 自动播放受 iOS/Android 音量、静音、音频焦点、蓝牙和系统中断影响，需要真机提示与恢复逻辑。
- TTS 可以独立失败；此时最终原文/译文仍可用，但当前没有独立 TTS 重试端点，客户端不应展示无效的“重试语音”操作。
- Redis 的内存回退只适合单进程开发；生产多实例必须使用 Redis adapter。
- Conversation/Subject 实时广播已按目标串行并对短暂 Redis `fetchSockets` 失败有限重试，强制断开失败也会立即处理本机 Socket；但尚无持久化 realtime event outbox。进程在数据库提交后、广播前崩溃时，好友/邀请/人员状态事件可能遗失；权限和消息仍以 PostgreSQL 为权威，客户端必须在重连时通过 REST/`room.joined` 重建快照。需要“必达”的社交通知在正式发布前仍需 transactional outbox/worker 或推送补偿。
- ARTC 凭证在签发时返回并可校验服务端最终 `mediaType`，但当前供应商 token 格式没有轨道级发布/订阅权限，已签发 token 也不会随本地通话 END 立即撤销。服务端会按通话/账号/设备授权、记录不含 token 的签发审计日志，并以双侧心跳收敛数据库状态；但恶意改造客户端、token 在有效期内重放及供应商层媒体权限仍是发布前必须验证/接受的残余风险。在客户端实现 token 刷新回调前，不能盲目把 TTL 降到 5 分钟导致长通话中断。
- `AUDIO_STORAGE_DRIVER=local` 只适合开发；生产配置会拒绝 local 并强制私有 S3。在线删除失败已有持久任务、租约和退避重试，未提交 TTS 也进入该队列，但当前对象 key 未区分 provisional；真实 bucket 的全 TTS 保留期或独立 provisional 前缀、长时间故障和备份传播仍未演练。
- 房间码便于输入但不是唯一凭证，仍需身份、状态和权限验证。
- 短期签名 TTS URL 默认 15 分钟过期；客户端需重新获取 messages 或 `room.joined` 数据取得新 URL。下载同时需要当前 Bearer Token，并在每次请求中反查所属会议权限；被移出、会话撤销或历史授权到期会立即阻止旧 URL，签名 TTL 只是第二层时限。

## 5. 法律与运营待定

正式上线前必须确定并写入政策：

- 运营主体、隐私/客服联系方式、适用法律和争议解决地。
- 服务器、阿里云和对象存储地域以及是否发生跨境处理。
- 账号、会议、TTS、审计、备份和删除请求的具体保留期限。
- 注册用户注销时，其参与但不拥有的会议之删除/匿名化规则。
- 真实启用的第三方 SDK、推送、监控和崩溃服务清单。
- 收费、免费额度、企业授权与退款规则。

未经补齐的政策模板不能直接作为正式法律文本发布。

## 6. 不得用以下结果替代验收

- mock provider 的固定文本不能证明翻译准确率。
- 生产已拒绝 mock，但“启动保护存在”仍不能替代真实阿里云账号验收。
- 单元测试不能替代 Android/iPhone 真机录放音。
- 模拟器扫码/链接不能替代正式域名系统验证。
- Debug 包不能替代 Play internal track 或 TestFlight 包。
- 单 API 实例不能证明 Redis 跨实例广播。
- “接口返回 200”不能证明客户、会议和消息不串数据。
- 删除在线记录不能证明对象存储、第三方和备份删除传播。

完整关闭证据记录在 [测试报告](./TEST_REPORT.md)。
